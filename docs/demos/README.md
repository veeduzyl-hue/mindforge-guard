# Buyer Demo Series

MindForge Guard is a verifiable runtime governance product for AI coding and release workflows.

This demo series gives buyers, evaluators, and GitHub visitors a simple way to understand what Guard does in practice:

- **see the action**
- **see the drift**
- **prove the governance**
- **understand the edition boundary**

Together, Demo A, Demo B, and Demo C form one buyer journey.
They do not introduce new product behavior.
They package the existing Guard surface into a more usable commercial story.

## The Buyer Journey

### Demo A: See the action

Start with the baseline question:

"What did the AI actually change, and can we inspect that before it turns into downstream release risk?"

Demo A shows how Guard exposes a visible governance surface around AI-assisted change using the current Community-safe workflow.

### Demo B: Prove the governance

Once the team wants more than change visibility, the next buyer question is:

"Can this release process produce bounded evidence and readiness-style governance artifacts that we can actually verify?"

Demo B shows the release-governance side of Guard using the existing evidence package, manifest, and final-acceptance semantics line.

### Demo C: Understand the edition boundary

After the buyer understands the workflow value, the next practical question is:

"What do we get in Community, what changes in Pro, and what justifies Pro+?"

Demo C answers that with the current released edition boundary, without changing any SKU promises.

## What This Series Does Not Change

This demo series preserves the current Guard philosophy and release boundary:

- additive-only
- recommendation-only
- non-executing
- default-off where applicable
- no hidden takeover
- no authority expansion

It does not turn Guard into:

- an execution-control platform
- a hidden enforcer
- a control plane
- a dashboard-first product
