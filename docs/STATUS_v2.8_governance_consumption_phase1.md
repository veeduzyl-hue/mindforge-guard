## Guard v2.8 Governance Consumption Phase 1

### Objective
- Start Governance Consumption Preparation v1 on top of the existing governance surface.
- Define a consumer-facing governance consumption profile without adding a new governance artifact.

### Included
- governance consumption profile constants and export boundary
- required vs optional consumer artifact profile
- support-only artifact classification for internal governance surfaces
- consumer-safe linkage profile for future consumer dependency freezing
- verification for governance consumption profile stability

### Explicitly not included
- no new governance object
- no second consumer
- no audit main-output mutation
- no audit verdict mutation
- no permit gate semantic change
- no governance artifact semantic change
- no drift / snapshot / risk integration
- no full enforcement platform

### Phase 1 Position
- governance consumption preparation remains additive-only
- existing runtime behavior and artifact semantics remain unchanged
- the future-consumer dependency profile is now explicit and verifiable
