# MindForge Guard Main Site Release Announcement v1

## Announcement

MindForge Guard is now available for commercial purchase.

MindForge Guard is runtime governance for AI-assisted execution: a local way for teams to make AI-assisted work visible, verifiable, and reviewable before release.

## Short launch copy

Ship AI-assisted work with release trust you can verify.

AI accelerates execution. Guard gives teams the missing governance surface: execution visibility, local verification, and reviewable evidence before work reaches release.

## Why now

AI-assisted execution is becoming normal in engineering workflows, but many teams still lack a practical way to inspect what happened, prove governance readiness, and compare change before publication.

Guard gives those teams a bounded path to release trust without forcing a hosted control plane into the main path.

## What buyers can do next

- Review AI-assisted changes before they become release drift.
- Prove release governance readiness with concrete evidence.
- Compare Community, Pro, and Pro+ in a practical buyer scenario.
- Buy a commercial edition through the Licensing Center.
- Download a signed license from the Customer Portal and install it locally.

## Demo-forward evaluation path

1. Demo A: Review AI-assisted changes before release.
2. Demo B: Prove release governance readiness.
3. Demo C: Compare editions in practice.

## Pricing and purchase path

Community remains the local baseline. Pro and Pro+ provide commercial access for paid analytics depth. Enterprise remains contact-led for procurement and rollout planning.

Commercial purchase flows through the MindForge Guard Licensing Center / Customer Portal. The Guard CLI remains local and verifies the signed license file locally.

## Boundary statement

MindForge Guard remains recommendation-only, additive-only, non-executing, and local-first. Licensing supports purchase and delivery without changing Guard CLI semantics.

This release copy does not change `audit`, `permit`, or `classify` behavior.
