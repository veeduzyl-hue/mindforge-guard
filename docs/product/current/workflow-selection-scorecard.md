# Workflow Selection Scorecard

## Purpose

Help a design partner choose the first trial workflow that is most likely to produce usable review evidence and clear adoption feedback.

## Good First Workflow Criteria

Good first workflows are:

- easy to describe in plain language;
- backed by visible evidence;
- safe to summarize with synthetic, redacted, or summary material;
- reviewed by a clear human owner;
- limited in external side effects;
- single-agent and recommendation-oriented.

## Poor First Workflow Criteria

Poor first workflows are:

- hard to explain without internal tribal knowledge;
- dependent on secrets or private records that cannot be redacted;
- broad enough to mix multiple workflows together;
- operationally sensitive or side-effect-heavy;
- missing a real human reviewer;
- multi-agent or execution-authoritative.

## Scorecard

| Criterion | Score 0 | Score 1 | Score 2 |
|---|---|---|---|
| Workflow is easy to describe | No | Partly | Yes |
| Evidence is available | No | Some | Clear |
| Sensitive data can be redacted | No | Partly | Yes |
| Human reviewer exists | No | Informal | Clear owner |
| External side effects are limited | High | Some | None or review-only |
| Workflow is single-agent | No | Mixed | Yes |

## Recommendation

- 8-10: good first workflow
- 5-7: possible but needs scoping
- 0-4: not a good first trial

This scorecard does not represent approval.

It does not judge safe-to-deploy status.

It only helps select a first trial workflow.

## Boundary

This scorecard helps with trial workflow selection only. It does not approve, block, deploy, certify, guarantee legal compliance, or control execution.
